# План экспериментального исследования

Прототип используется для проверки автоматизированной методики оценки и оптимизации производительности клиентской части веб-приложения.

## Измеряемые маршруты

- `/dashboard` — панель мониторинга технологического процесса;
- `/events` — журнал событий;
- `/reports` — отчеты по результатам измерений;
- `/settings` — пороговые значения performance budget;
- `/details/line-a` — карточка контролируемого объекта.

## Последовательность эксперимента

1. Выполнить production-сборку frontend-приложения.
2. Запустить Vite preview server.
3. Выполнить Lighthouse CI для каждого маршрута.
4. Сделать 5 прогонов для каждого маршрута.
5. Получить HTML- и JSON-отчеты Lighthouse.
6. Рассчитать медианные значения метрик.
7. Сравнить значения с performance budget.
8. Выполнить отдельную проверку `budgets.json` по ресурсам и ключевым метрикам.
9. Сформировать Markdown/JSON-сводку для таблиц диссертации.
10. Сравнить исходную и оптимизированную версии.

## Измеряемые метрики

- Lighthouse Performance Score;
- FCP;
- LCP;
- TBT;
- CLS;
- Speed Index;
- общий объем переданных ресурсов;
- количество сетевых запросов.

## Экспериментальные варианты

- исходная версия: URL содержит `?variant=baseline`, что включает контролируемую нагрузку на основной поток и дополнительную DOM-нагрузку;
- оптимизированная версия: обычные URL без query-параметра.

Такой подход позволяет воспроизводимо показать проблему исходной версии и эффект оптимизации без ручной подмены проекта.

Для демонстрации версий в Git подготовлены отдельные ветки:

- `baseline-version` — исходная версия с включенными деградациями;
- `optimized-version` — оптимизированная версия, соответствующая основной ветке `main`.

## Виды контролируемых деградаций

| Деградация в исходной версии | Метрика | Оптимизация |
| --- | --- | --- |
| Синхронная вычислительная нагрузка на main thread | TBT | устранение блокирующего участка |
| Дополнительный тяжелый JS-модуль | JS transfer, total transfer | исключение модуля из оптимизированного маршрута |
| Дополнительная SVG-схема процесса | image transfer, total transfer | исключение тяжелого изображения из оптимизированной версии |
| Дополнительные запросы ресурсов | request count | сокращение набора загружаемых ресурсов |

## Режимы контроля

- `lighthouserc.cjs` — рабочий режим с предупреждениями, который формирует отчеты даже при отклонениях;
- `lighthouserc.baseline.cjs` — режим исходной версии с контролируемыми деградациями;
- `lighthouserc.strict.cjs` — строгий режим, где превышение ключевых порогов приводит к ошибке проверки.
